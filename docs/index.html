Why SQLSense
Every guard you need.
Zero friction.

Existing MCP database tools give agents raw access. SQLSense wraps every query in a safety layer built for production environments.

🚫
Guardrails Engine

Blocks DROP, TRUNCATE, DELETE without WHERE, writes in readonly mode, and SQL injection attempts before they reach your database.

📋
Full Audit Trail

Every query — allowed or blocked — written to JSONL. Stream to Splunk, Datadog, or CloudWatch. Know exactly what your agents did.

🔢
Auto-LIMIT

SELECT without LIMIT? SQLSense automatically caps it. Configurable max rows per query. Protects against full-table scans at 3am.

🙈
Column Blocklist

password, ssn, credit_card, api_key — blocked by default. Add your own sensitive columns. Agents never see what they shouldn't.

🗄️
Multi-Database

SQLite, PostgreSQL, SQL Server, Snowflake. One interface, consistent guardrails everywhere. MySQL and BigQuery connectors in progress.

⚡
MCP Native

Speaks MCP 2024-11-05. Works with Claude Desktop, Claude Code, and any MCP-compatible agent framework out of the box.

Architecture
How it works

Five steps between your AI agent and your database.

1
Agent calls tool

Claude or any MCP agent calls sql_query with a SQL string

2
Guardrails check

Query parsed and checked against all rules — risk assessed

3
Rewrite or block

Safe queries get LIMIT added. Dangerous ones return an error

4
Execute

Allowed queries run against your database via the connector

5
Audit log

Every query — allowed or blocked — written to JSONL immediately

Get started
Up in 60 seconds.

Zero mandatory dependencies. Add database drivers only for what you need.

Install
# Base install (SQLite built-in)
pip install sqlsense

# With PostgreSQL support
pip install "sqlsense[postgres]"

# With SQL Server support
pip install "sqlsense[sqlserver]"

# Everything
pip install "sqlsense[all]"
Claude Desktop Config
{
  "mcpServers": {
    "sqlsense": {
      "command": "sqlsense",
      "args": [
        "serve",
        "--dsn",
        "postgresql://user:pass@localhost/db"
      ]
    }
  }
}
CLI usage
# Start MCP server
sqlsense serve --dsn "postgresql://..."

# Check a query before deploying
sqlsense check "DELETE FROM users"
🚫 BLOCKED (risk: HIGH)

# View audit log
sqlsense audit --tail 50

# Allow writes (use with care)
sqlsense serve --dsn "..." --allow-writes

# Block specific tables
sqlsense serve --dsn "..." \
  --block-table secrets --block-table audit_log
